# Journals

This directory contains individual contribution logs for each team member.

Each journal documents:

* Personal contributions and activities
* Work performed on the project
* Specific sections of reports worked on (Overview, Design, Architecture)

**Update Frequency:** Every 2 weeks (or more frequently if needed)

---

## Guidelines for Entries

### What to Include

- **Activities:** Specific tasks completed (e.g., "Analyzed dependencies using tool X", "Identified 3 design patterns")
- **Contributions:** Which report sections you worked on and what was added/modified
- **Be Concise:** One sentence per activity is sufficient

### Entry Format

Use this template each time you update your journal:

```markdown
### [YYYY-MM-DD]

## **Activities:**

* [What you did]
* [What you did]

**Contribution to reports:**

* Overview: [What you added/modified, or N/A]
* Design: [What you added/modified, or N/A]
* Architecture: [What you added/modified, or N/A]

---
```

### Example

```markdown
### 2026-04-18

## **Activities:**

* Analyzed code dependencies using static analysis
* Identified Singleton pattern in Logger configuration

**Contribution to reports:**

* Overview: Added code statistics
* Design: Added dependency analysis section
* Architecture: N/A

---
```

## Summary Section

At the end of your journal, add a **Summary of Contributions** with a brief overview of your total contribution to each report:

```markdown
## Summary of Contributions

* **Overview:** [Your main contributions]
* **Design:** [Your main contributions]
* **Architecture:** [Your main contributions]
```

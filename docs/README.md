# Reports

This directory contains the three main reports required for the project:

* **overview.md**
  Provides a high-level description of the system, including:

  * Purpose and stakeholders
  * Basic statistics (e.g., number of files, lines of code, modules, developers)

* **design.md**
  Focuses on software design aspects:

  * Code dependencies (based on imports)
  * Knowledge dependencies (based on co-change in commits)
  * Identification and analysis of at least four design patterns

* **architecture.md**
  Describes the system architecture using the C4 model:

  * Context diagram
  * Container diagram
  * Component diagrams
    Includes discussion of architectural characteristics and potential violations of SOLID principles

All reports are written in Markdown and respect the required word limits.
